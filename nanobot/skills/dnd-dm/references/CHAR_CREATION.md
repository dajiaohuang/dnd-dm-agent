# 角色创建流程（对话版）

---

## 概述

当玩家选择「开新游戏」时，以对话交互的方式引导玩家创建角色。

**核心原则：**
- 逐项提问，每次只问一个选择
- 展示选项时保持清晰、简洁
- 2024 版规则优先
- 每完成一项做一次确认

**阶段顺序：**
1. 属性生成（核心——决定职业方向）
2. 种族选择
3. 职业选择
4. 背景选择
5. 装备/法术/技能选择（按职业不同）
6. 角色命名与定型
7. 最终摘要

---

## 阶段 1：属性生成

### 1.1 选择生成方式

```
📜 你的命运由骰子裁定——选择力量的来源：

  1️⃣ 掷骰 —— 4d6 取最高3次(4d6dl1)，卷起命运的衣角
  2️⃣ 标准数组 —— 15/14/13/12/10/8，交给均衡
  3️⃣ 购点法 —— 27点预算，你的意志塑造每一点
```

当玩家选择后，执行对应方法。

### 1.2 掷骰执行

**掷骰（4d6dl1 x6）：**  
城主后台掷 6 组 4d6dl1，展示结果：

```
🎲 命运之骰掷下：
  ┌─────────────────────────────┐
  │ [1] 16 (6+5+5 ｜ 弃1)      │
  │ [2] 13 (5+4+4 ｜ 弃3)      │
  │ [3] 10 (4+3+3 ｜ 弃2)      │
  │ [4] 15 (6+5+4 ｜ 弃3)      │
  │ [5] 12 (5+4+3 ｜ 弃2)      │
  │ [6] 9  (4+3+2 ｜ 弃1)      │
  └─────────────────────────────┘

❓ 将哪个数值分配给 [力量/敏捷/体质/智力/感知/魅力]？
```

分配规则：每次选一组数值→选一个属性，直到 6 组全部配完。

**标准数组：** 直接展示 15/14/13/12/10/8，逐项分配。

**购点法（27点）：** 按 2024 版购点表操作：

```
🏛️ 购点预算：27/27
剩余点数：27

当前属性（全为 8，消耗 0 点）：
  力量  8 (+0)  │  敏捷  8 (+0)  │  体质  8 (+0)
  智力  8 (+0)  │  感知  8 (+0)  │  魅力  8 (+0)

❓ 选择你想提升的属性：
  1️⃣ 力量  2️⃣ 敏捷  3️⃣ 体质
  4️⃣ 智力  5️⃣ 感知  6️⃣ 魅力

『当前选择：[属性名]，每次+1，当前8→9（消耗1点）』
```

购点表（2024版）：8→0, 9→1, 10→2, 11→3, 12→4, 13→5, 14→7, 15→9

**手动录入：** 玩家逐个输入 6 个属性值。

### 1.3 属性最终表格

所有属性分配完毕后展示：

```
📊 基础属性
━━━━━━━━━━━━━━━━━━━━━━━
  力量  16  (+3)  │  敏捷  10  (+0)
  体质  15  (+2)  │  智力  13  (+1)
  感知  12  (+1)  │  魅力  9   (-1)
━━━━━━━━━━━━━━━━━━━━━━━
```

## 阶段 5：职业特定选择

根据所选职业不同：

- **法师/术士/邪术师/吟游诗人/游侠/德鲁伊/牧师/圣武士** → 选择法术
  - 已知法术列表展示
  - 每次选 1 个，直至达到等级允许数量

- **所有职业** → 确认初始装备
  - 按 2024 PHB 装备包选择
  - 展示装备选项 2-3 行

```
🏛️ 角色定型 ──── [角色名]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  🏷️  种族：[种族]
  ⚔️  职业：[职业] Lv.1
  🎭  背景：[背景]
  ─────────────────────────────────
  💪 力量  16 (+3)    🧠 智力  13 (+1)
  🤸 敏捷  10 (+0)    🧘 感知  12 (+1)
  🛡️ 体质  15 (+2)    💎 魅力  9  (-1)
  ─────────────────────────────────
  ❤️ HP：[职业初始HP]        🛡️ AC：[计算值]
  ─────────────────────────────────
  🎯 熟练加值：+2
  ─────────────────────────────────
  📋 熟练技能：[背景2+职业2+...]
  ⚔️ 初始装备：[简要列]
  ⚡ 法术：[如有]
  ─────────────────────────────────
  📖 背景特质：[简要]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❓ 确认创建？[是/否]
```

确认后，必须先用确定性 CLI 将完整角色卡写入当前 `campaign_id` 的数据库队伍与角色表：

```powershell
python -m nanobot.dnd.db.cli character create `
  --campaign <campaign-id> `
  --name "<角色名>" `
  --player "<玩家名>" `
  --class "<职业>" `
  --level <等级> `
  --hp <当前HP> `
  --max-hp <最大HP> `
  --ac <AC> `
  --sheet-file "<UTF-8 JSON角色卡路径>"
```

命令成功返回 `character_id` 后，再创建初始或剧情节点 Snapshot。禁止只把角色卡写入
会话、记忆或 workspace JSON 后便声称创建完成。用下面的命令核验当前战役角色：

```powershell
python -m nanobot.dnd.db.cli character list --campaign <campaign-id>
```

## 协作规则

1. **城主骰种子，玩家选分配**：城主后台模拟骰子结果，玩家只做定向分配（不疑心——一且信任城主的骰子）。
2. **每次只问一个选择**：不问并行问题。阶段内串行推进。
3. **确认机制**：关键决定后（种族/职业/背景）做一次简单确认，允许回退一步修改。
4. **取消/退出**：任一阶段玩家可说"取消"退出创建流程。
5. **改点窗口**：最终摘要展示时允许重新分配 1-2 个属性值（不是全部重来），或完全重来。

## 快捷通道（进阶玩家）

以上全流程对新玩家友好。对老手可直接提问：

```
❓ 熟悉过程？[是] ≥ 直接填写摘要，城主生成最终卡
   [否] ≥ 进入全流程引导
```

跳过阶段 2-6 的逐项引导，直接让玩家提供：
`[种族] [职业] Lv.1 | [背景] | [属性: 16/15/13/12/10/9]`

城主一键生成最终摘要。
