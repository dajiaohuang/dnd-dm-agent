# 地图子系统（Excel核心系统）

## 总则
本系统是城主运行的核心工具之一，用于在玩家进入新场景时提供可视化的空间定位。

**工具载体：** Microsoft Excel
**格式标准：** .xlsx 文件，通过飞书发送
**文件存放：** `工作目录\maps\` 目录
**文件命名：** `场景名.xlsx`（如`浴室庭院D1.xlsx`、`浴池大厅D2.xlsx`）

---

## 一、进入新场景的完整流程

```
步骤1 | 玩家表示要进入新场景/房间
        ↓
步骤2 | 城主读取模组该场景描述（尺寸、物品、NPC、门、暗门、陷阱、容器等）
        ↓
步骤3 | 城主生成Excel地图文件并发送给玩家
        ↓
步骤4 | 城主用文字描述场景环境（玩家看到什么）
        ↓
步骤5 | 玩家自主选择行动
        ↓
步骤6 | 当前场景结束后 → 玩家通过"问队友"触发下一阶段
```

**禁止：**
- 进入新场景后直接触发关键节点，不给探索空间
- 跳过地图生成步骤
- 玩家没要求查看环境时提前生成地图

## 二、地图制作标准

### 2.1 网格标准
- **每格 = 5尺**（DND标准战斗网格尺寸）
- 列宽：3.5（保证正方形格子）
- 行高：28（与列宽配合，确保格子为正方形）
- 坐标轴：横轴字母A-Z，纵轴数字1开始
- 字体：Segoe UI Emoji（确保emoji正常渲染）
- 对齐：居中对齐

### 2.2 着色标准
| 元素 | 颜色代码 | 用途 |
|------|---------|------|
| 墙壁 | 0x404020 | 外墙、室内分隔墙、石柱 |
| 地板 | 0x1A0D0D | 可通行的室内/室外地面 |
| 水域 | 0x4A2010 | 水池、喷泉、河流、下水道 |
| 家具 | 0x2A1820 | 柜子、桌椅、床铺等 |

### 2.3 Emoji图例标准
| Emoji | 含义 |
|-------|------|
| 🧝 | 老白干一杯 |
| 🛡️ | 伊索尔德·铁誓 |
| ⚕️ | 莉莉安·晨祷 |
| 🔮 | 科林·霜风 |
| 🚪 | 门（关闭） |
| 🔓 | 门（打开/未锁） |
| 💧 | 水/水池 |
| 🪟 | 窗户 |
| 🪨 | 石柱/石块/障碍物 |
| 🔲 | 更衣柜/储物柜 |
| 🗡️ | 敌人（战斗时标注） |
| 📦 | 箱子/容器 |
| 🔑 | 可交互物品/机关 |
| 🔥 | 火源/火炬 |
| 🪜 | 楼梯 |
| 🕯️ | 灯光/蜡烛 |
| 🪦 | 墓碑/石棺 |
| ⬛ | 缝隙/洞口 |
| ❓ | 未探明的异常 |

### 2.4 地图必须标注的内容
- 房间四周边界墙
- 所有门的位置（🚪）
- 主要家具/地貌位置
- 所有可交互物品位置（📦、🔑等）
- 所有NPC/敌人的位置（如果可见）
- 队伍四人的当前位置
- 坐标轴
- 图例

## 三、地图更新规则

### 3.1 触发更新的情况
玩家与地图元素交互后导致状态变化，必须立即更新并重新发送。包括但不限于：
- 打开/关闭一扇门
- 移动或取走一个物品/容器
- 触发陷阱后状态变化
- 暗门被发现并打开
- 战斗后怪物尸体/障碍物变化
- 水池水位变化（放水/堵水等）
- 墙壁被破坏或通道被打通
- 角色移动到新位置（战斗时每轮更新）

### 3.2 更新方式
1. 在原有Excel文件上直接修改（不创建新文件）
2. 保存后通过飞书重新发送
3. 发送时附带简要说明标注了哪些变化

## 四、未来优化预留接口

以下为地图子系统预留的扩展方向，当玩家提出相关需求时，可据此方向进行优化：

### 4.1 模板化/自动化预留
- 未来可创建**Excel宏/模板**，实现一键生成基础结构（房间框架、坐标轴、着色）
- 未来可将常用房间模板（方形房间、L形走廊、T形路口等）预设为模板文件
- 未来可考虑使用Python脚本通过`openpyxl`库自动生成Excel文件（减少手动操作）

### 4.2 交互方式扩展预留
- 未来可让**玩家通过飞书指令更新自己的位置**（如"老白移动到F7"），城主确认后更新地图
- 未来可支持**多楼层地图**（每个楼层一个sheet页）
- 未来可考虑在地图中**标注视线范围**（阴影区域/视野盲区）

### 4.3 样式扩展预留
- 未来可根据需要增加新的着色代码（如陷阱区域用红色、魔法区域用紫色）
- 未来可增加**网格线样式**（实线/虚线表示不同的意义）
- 未来可增加**箭头标注**（表示移动路线、攻击方向等）

### 4.4 玩家自定义预留
- 玩家可要求调整格子尺寸（如改为10尺/格用于大地图场景）
- 玩家可要求在新地图中使用自定义emoji标注
- 玩家可要求记录已探索/未探索区域

---

## 五、地图子系统规则同样适用规则0.5

玩家在任何时候如果对地图系统有改进意见（无论是新场景的呈现方式、交互方式、还是视觉样式），都可以提出，城主将：
1. 记录优化需求
2. 更新本子系统中的对应规则
3. 以后按新规则执行
