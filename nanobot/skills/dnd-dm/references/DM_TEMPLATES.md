# DM 输出模板

模板只规定面向玩家的展示，不是权威数据结构。所有数值和状态来自当前 `campaign_id`
的数据库状态；所有随机结果来自 `dnd-engine`。

## 1. 场景

```text
📍 [地点]

[只描述角色能够感知的环境、人物、动静和明显出口。]

你们打算做什么？
```

不要展示房间编号、节点 ID、触发条件、隐藏门、陷阱或未发现敌人。

## 2. 检定

```text
🎲 [角色]进行[检定名称]
掷骰：[d20结果] + [公开加值] = [总值]
结果：[成功/失败及可观察后果]
```

优势或劣势时展示实际使用的骰子。隐藏 DC 在行动确认前不公开。

## 3. 攻击与伤害

```text
⚔️ [攻击者] → [目标]
攻击：[d20] + [攻击加值] = [总值]，[命中/未命中]
伤害：[伤害骰明细] = [总伤害]
[抗性、免疫、临时HP或状态变化]
```

## 4. 战斗概况

```text
⚔️ [战斗名称] · 第[轮次]轮
当前行动：[单位]

1. [单位] — HP [当前]/[最大] — [状态]
2. [单位] — HP [当前]/[最大] — [状态]

环境：[地形、光照、掩护和特殊效果]
```

对玩家隐藏不应公开的怪物精确数值，可用受伤程度代替。

## 5. 回合提示

```text
轮到[角色]。
可用：移动[剩余]、动作[可用/已用]、附赠动作[可用/已用]、反应[可用/已用]
你要怎么做？
```

## 6. 队伍概况

```text
👥 [队伍名称] · [地点]

- [角色] [职业] Lv.[等级] — HP [当前]/[最大] — AC [数值]
- [角色] [职业] Lv.[等级] — HP [当前]/[最大] — AC [数值]

共享资源：[金币及其他共享资源]
当前任务：[任务摘要]
```

## 7. 角色详情

依次展示：身份与等级、六维属性、HP/AC/速度、熟练项、攻击、职业与种族能力、法术与
法术位、装备、背包、持续效果和经验。不要省略会影响当前裁决的资源。

## 8. 任务

```text
📜 任务

进行中
- [任务]：[当前目标]

已完成
- [任务]

失败
- [任务]：[失败原因]
```

## 9. 战役列表

```text
🗺️ 活动战役

1. [名称] (`campaign_id`) — [模组] — [状态摘要]
2. [名称] (`campaign_id`) — [模组] — [状态摘要]
```

多个战役存在时让玩家明确选择，不按角色名或槽位猜测。

## 10. Snapshot 列表

```text
💾 [战役名称]的存档

1. 槽位 [slot] — [标签] — 第[章节]章 · [地点] · [时间]
2. 槽位 [slot] — [标签] — 第[章节]章 · [地点] · [时间]
```

只展示当前 `campaign_id` 的 Snapshot。

## 11. 保存成功

```text
💾 已保存
战役：[名称] (`campaign_id`)
槽位：[slot]
标签：[label]
位置：[章节/地点]
校验：[hash前12位]
```

## 12. 恢复成功

```text
↩️ 已恢复
战役：[名称] (`campaign_id`)
槽位：[slot]
位置：[章节/地点]
审计：[audit_id]
```

## 13. 升级

每次只处理一个步骤：HP → 能力/子职 → 熟练加值 → 技能与豁免 → 专长或属性提升 →
法术与法术位 → 装备和派生数值。最终展示完整变化清单，确认数据库提交和 Snapshot 槽位。

## 14. 章节过渡

```text
📖 [章节名称]完成

完成节点：[摘要]
未决线索：[摘要]
队伍状态：[摘要]
下一阶段：[升级/休整/进入下一章]
```

完成状态更新后、加载下一章前创建 Snapshot。
